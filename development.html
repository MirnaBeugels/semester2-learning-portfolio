Pinkpay

Toegepast: JavaScript/CSS/HTML

Voor mijn individuele project heb ik een digitale munten app bedacht voor Pinkpop. Om deze te kunnen gaan maken ben ik begonnen met het volgen van de PWA workshop van Michael. Daarna ben ik gestart met het proberen te bouwen van de app, op verschillende manieren. Ik heb eerst een hele tijd gedacht over of ik bijvoorbeeld een framework als Vue of React nodig zou hebben en daar ook even mee gespeeld. Echter vond ik dat nog erg moeilijk, omdat je dan niet alleen JavaScript aan het leren kennen bent maar ook het Framework. En voor je je weg in zo'n framework een beetje gevonden hebt wat betreft bijvoorbeeld de structuur van de bestanden ben je al een heel eind verder. Het idee van een framework heb ik dus laten varen, met name omdat ik ook eerst JavaScript in de basis beter wil leren begrijpen.

Ik ben daarna verder gegaan met het bouwen van de basis structuur van de pagina met de verspringende blokjes. Dat lukte allemaal nog prima.

Daarna heb ik gekeken naar het authenticatie gedeelte. Na wat zoekwerk kwam ik uit bij Firebase, de documentatie en bijbehorende instructies op de website van Firebase zijn wat moeilijk te begrijpen wanneer je kennis van JavaScript nog niet zo ver is. Dit komt doordat je Firebase op verschillende manieren kunt implementeren en je de instructies daarop moet aanpassen. Hierdoor was het al best een uitdaging om de authenticatie werkend te krijgen.

Na de authenticatie wilde ik met Firebase Realtime Database meer gegevens van gebruikers opslaan. Helaas kwam ik daar niet meer op tijd uit. Wel heb ik nog een tutorial gevonden die ik wil gaan volgen om Firebase beter te leren kennen en begrijpen wat er staat. Deze vindt je hier: Tutorialspoint.com/firebase.

Mijn plan is om op de pagina waar je naar doorlinkt na de authenticatie uit te lezen welke gebruiker er ingelogd is, dan in de database te checken of die gebruiker al bestaat. Zo niet, vragen om extra gegevens van de gebruiker zoals naam en email en deze op te slaan in de database. Bestaat de gebruiker wel al de intropagina weer te geven met een persoonlijke boodschap. In deze database wil ik ook bijhouden hoeveel munten een gebruiker heeft en de bestellingen opslaan.

Verder heb ik voor dit project SCSS gebruikt in plaats van reguliere CSS, een bewuste keuze omdat ik er benieuwd naar was omdat je het vaak ziet als een gewilde kennis bij vacatures. Om het te leren toepassen heb ik een tutorial gevolgd op Codecademy en instructies geraadpleegd op Sass-lang.com

Wat ik heb geleerd...

  • Wat een PWA is
  • Dat je geen ingewikkelde programmeertalen nodig hebt om een app te maken die niet OS gebonden is.
  • Hoe je Firebase kunt gebruiken om een database van gebruikers te maken

Tekst langs een pad animatie

Toegepast: JavaScript/CSS/HTML

Mijn eerste stap in SVG animaties was het maken van een animatie langs een pad. Om te beginnen heb ik met het pentool in Illustrator mijn naam geschreven, met alle letters aan elkaar, en deze als een SVG opgeslagen. Daarna heb ik deze SVG code in mijn HTML geplakt. En middels CSS zo geanimeerd dat het lijkt alsof deze geschreven wordt.

Wat ik heb geleerd...

Dat je het kan laten lijken of een pad getekend wordt door gebruik te maken van een stippellijn. De stippel (stroke) en de witruimte (dash) maak je net zo lang als het hele pad is. De lengte van het pad kun je met JavaScript berekenen. Door de offset naar 0 te animeren met een css transitie wordt de streep steeds meer zichtbaar totdat je het hele pad kunt zien.

Memory

Toegepast: JavaScript/CSS/HTML

Om JavaScript beter te leren kennen heb ik een Memory spel gemaakt. Wanneer je op de startknop drukt begint de tijd te lopen. Vervolgens klik je op twee kaarten en checkt het spel of die twee kaarten dezelfde afbeelding hebben. Wanneer dat niet zo is draaien beide kaarten weer om. Het spel houdt bij hoeveel kaarten je al omgedraaid hebt. Wanneer je alle koppels gevonden hebt geeft het spel aan dat je gewonnen hebt. Ook geeft het in seconden aan hoe lang je daarover gedaan hebt en hoeveel beurten je nodig had om te winnen (aantal omgedraaide kaarten gedeeld door 2).

Wat ik heb geleerd...

  • Hoe je door een array met afbeeldingen kunt loopen voor een x aantal keren (zo vaak als het aantal kaarten op het bord gedeeld door 2), waarbij iedere keer een willekeurig item uit de array gekozen wordt en aan het eind van de loop alle willekeurige items gereturned worden waarna ze in een willekeurige volgorde gezet worden zodat iedere keer dat je de pagina laadt de kaarten op een andere plek staan.
  • Hoe je 2 items waarop geklikt wordt een nieuwe css klasse kunt geven, waarmee je de afbeeldingen waarop geklikt is zichtbaar maakt. Hoe je vervolgens deze twee afbeeldingen met elkaar vergelijkt en bekijkt of ze dezelfde url hebben. Hoe je ze wanneer dit waar is weer een nieuwe klasse meegeeft die aangeeft dat ze matchen. Hoe je kaarten die niet matchten vervolgens weer omdraait en de klasse die aangaf dat ze omgedraaid waren weer verwijderd zodat je het opnieuw kunt blijven proberen totdat je alle kaarten die matchen gevonden hebt.
  • Hoe je de startknop inactief maakt zolang nog niet alle kaarten omgedraaid zijn en het spel nog loopt.
  • Hoe je kunt bijhouden hoe lang een spel duurt door te beginnen bij 0, de tijd te gaan laten lopen vanaf het moment dat er op de startknop of een eerste kaart gedrukt wordt. De tijd laat stoppen wanneer alle kaarten gematcht en omgedraaid zijn. Weergeeft wat de totale tijd was. En de tijd weer op 0 zet en opnieuw laat starten met tellen wanneer er op de startknop gedrukt wordt.

IXD Lectoraat Header

Toegepast: JavaScript/CSS/HTML

Voor de IXD lectoraat website moest een header en layout gemaakt worden die als basis voor alle andere pagina's kon dienen. Op deze manier zouden we ervoor kunnen zorgen dat ondanks dat iedere pagina door iemand anders gemaakt werd, deze toch hetzelfde uiterlijk hadden. De header heeft een slider met 3 afbeeldingen die middels een setInterval steeds na 7 seconden wisselt. Ook beschikt de pagina over navigatie bar die sticky is en aan de bovenkant van de pagina blijft plakken wanneer je naar beneden scrollt. Verder heeft het content blok een negatieve margin aan de bovenkant, zodat deze de achtergrond overlapt.
